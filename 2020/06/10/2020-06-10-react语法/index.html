jsx语法以及规范

  • 响应式直接操作数据，尽量不做dom操作
  • 就是在js里面写html标签
  • jsx中不希望自动转译标签 需要在标签上写入 <li dangerousSetInnerHTML={{__html:item}}></li>
  • state 不允许直接修改 immutable
  • 定义数据使用this.state
  • 组件定义第一个字母必须使用大写
  • 渲染需要使用花括号包裹，
  • 绑定时间需要使用.bind(this)重新定义this指向
  • 方法里改变数据需要使用this.setState({}),不能直接修改state里面的数据
  • 注释需要在花括号里面填写{ // jkh} {/**/}
  • 样式使用classNumber，不能使用class ，class在react中是一个关键字
  • 在jsx中循环操作数据需要在渲染的那条数据中添加 key=""值
  • 把this指向的代码放在constructor里面
  • 所有的元素必须放在一个元素里 <div> 也可以使用占位符,fragment
  • 声明式开发（减少dom操作量）
  • 可以与其他框架并存
  • 组件化
  • 单向数据流（父组件给子组件传值不能修改）
  • 视图层框架（需要配合其他的，只能做视图）
  • 函数式编程（有利于自动化测试）
  • label标签的作用扩大点击范围，使用标签接收id='label定义的htmlFor值’
  • 使用label标签不能直接使用for定义，需要htmlFor来定义 <label htmlFor="e">输入内容</label>

父子相互传值

  • 父组件向子组件传递数据 属性的形式传递 子组件prop接收
  • 子组件获取父组件值，在子组件里定义方法

    props state render 函数的关系

  • 当组件的state或者props发生改变的时候，render函数也会重新执行
  • 父组件的render重新执行，子组件render也重新执行

react中虚拟dom

虚拟DOM就是js对象，用它来描述真是DOM

底层使用React.createElement()方法生成虚拟DOM（js对象）在转换成真实的DOM

React.createElement('idv',{},'item')

虚拟DOM中diff算法

虚拟DOM比对（同层对比）时使用diff算法

优点

  • 性能提升了
  • js对比js降低了性能的消耗
  • 它使得跨端应用得以实现

ref使用

直接获取操作dom ref={(input) => {this.input = input}}

生命周期函数

是指在某一刻组件会自动调用执行的函数

  • initialization：组件初始化加载
  • Mounting（挂载）
    • ComponentReceiveProps：只有在子组件接受父组件参数，父组件render执行，他就会执行，并且在第一次存在父组件中不会执行
    • componenWillNount：组件挂载到界面之前执行
    • render：界面渲染
    • componenDidNount：组件挂载到界面之后执行
  • Update（更新）
    • componenWillReceProps
    • shouldomponenUpdate：组件被更新之前，该函数要返回一个bloon值，return false该组件不会被更新
    • shouldomponenUpdate：组件被更新之前
  • Unmounting(去除)
    • conponentWillUnmount
